Design Critique: FlipIQ Deal Dashboard (Funding Pipeline)
Date: March 24, 2026
Context: New "Deal Dashboard" tab added to FlipIQ USER COMMAND — a CSM tool used daily by Ramy to monitor 21 Acquisition Associates across 5 organizations.
Stage: Early implementation — recently added, not yet in the spec document.
Primary User: Ramy (CSM Lead), with visibility for Tony (Product), Ethan (Accountability), and the COO.

Overall Impression
The deal dashboard has strong bones — the 5-stage pipeline cards, revenue forecast chart, and AA-grouped deal table are the right data to show. The biggest opportunity is reducing cognitive load. Right now, the tab dumps a lot of raw numbers without guiding Ramy toward the one thing she needs to answer each morning: "Which AAs are stuck in the pipeline, and where?" The dashboard currently shows deals, but doesn't yet tell a story about deal velocity or risk.

2. Visual Hierarchy
What draws the eye first: The 5 pipeline stage cards — their colored backgrounds and large count numbers pull attention correctly. This is good.
Reading flow: Eye moves from pipeline cards → forecast chart → breakdown panels → deal table. The problem is the three breakdown panels (By Source, By Property Type, By Intent) interrupt the natural flow from "pipeline overview" to "which deals need attention." They're analytical context, not action drivers.
Recommended change: Move the three breakdown panels into a collapsible "Analytics" section below the deal table. Most mornings, Ramy doesn't need source/type/intent breakdowns — she needs pipeline stage → stuck deals → action. Reserve the analytics for weekly reviews when Tony or Ethan want to slice the data.
Emphasis issues:

The pipeline stage cards treat all 5 stages equally (same size, same visual weight). But "In Negotiations" and "Offer Accepted" are where deals stall. Consider making these two cards slightly larger or adding a "velocity" indicator (avg days in stage) to call out bottlenecks.
Commission totals are prominently shown, but Ramy's primary metric is deal count (2/month/AA), not dollar amounts. Lead with count, show dollars secondary.

4. Accessibility

Color contrast: The 7-8px text in #94A3B8 on white backgrounds fails WCAG AA contrast ratio (4.5:1 required for small text). This affects the subtitle text, column headers, and percentage labels throughout the deal tab.
Touch targets: The filter toggle buttons are ~24×20px. WCAG recommends 44×44px minimum. On a laptop touchpad or if this is ever viewed on a tablet, these are hard to hit.
Text readability: The 7px font in the expanded deal detail rows is below the generally recommended 12px minimum for body text on screen. Even for data-dense tables, 10px is the practical floor.


5. What Works Well

Pipeline-as-funnel visualization with the 5 stage cards and arrow connectors is exactly the right mental model for a funding pipeline. It maps to how AAs think about their work.
Cross-filtering between the main dashboard filters (Org, Phase, Health) and the deal-specific filters (Source, Type, Intent) is powerful. Ramy can ask "Show me all red-health AAs at TD Realty with MLS deals" in 3 clicks.
Revenue forecast chart with hover tooltips showing deal count and value per period is well-executed. The data is there — it just needs a target line to become actionable.
Expandable AA rows in the deal table are the right pattern for progressive disclosure. Show the summary, click to see deal-level detail.
Commission calculation logic (different rates for Wholesaler vs MLS vs Off Market) being built into the data model means the numbers Ramy sees are real, not estimates.


6. Priority Recommendations
1. Add a "Pipeline Health" KPI strip at the top (High Impact, Low Effort)
Why: The dashboard currently opens with a title and subtitle of aggregate numbers. Ramy's first question every morning is "Who's behind?" not "What's the total pipeline?" A 3-metric KPI strip — (a) AAs on pace for 2 deals/month, (b) deals stuck 7+ days in a stage, (c) biggest bottleneck stage — turns the tab from a data dump into a morning briefing.
How: Add a horizontal card row above the pipeline stages. Three cards, each with a number, label, and color (green if healthy, red if concerning). Same pattern as the "Total AAs / Critical / At Risk / Draft Emails" strip on the Overview tab.
2. Add "Days in Stage" to the deal table and pipeline cards (High Impact, Medium Effort)
Why: The deal dashboard shows where deals are but not how long they've been there. A deal sitting in "In Negotiations" for 3 days is fine. A deal sitting there for 21 days is a problem. Without velocity data, the pipeline view is a snapshot that doesn't reveal momentum or stagnation.
How: Add a days_in_stage computed field to each deal. Show it as a column in the deal table with color coding (green <7d, yellow 7-14d, red 14d+). On the pipeline stage cards, show "Avg X days" beneath the deal count. This single addition transforms the deal tab from "here's what exists" to "here's what needs attention."
3. Show all pipeline stages in the deal table, not just closed deals (High Impact, Medium Effort)
Why: The current table only shows "Offer Accepted" and "Acquired" deals. That's the finish line, not the journey. Ramy needs to see which deals are stuck in Active Pipeline (no offer yet), which have offers out but no response, and which are in negotiation limbo. This is the coaching intelligence that drives her daily work.
How: Change the table filter default to "All stages." Add a stage filter dropdown. Add stage-colored badges to each row. Sort by "Days in stage" descending so stuck deals surface first.
4. Collapse the analytics panels and promote the deal table (Medium Impact, Low Effort)
Why: The "By Source / By Property Type / By Intent" panels take up significant vertical space between the pipeline overview and the deal table. For Ramy's daily workflow, these are secondary. Pushing the deal table — where she identifies stuck deals and takes action — below the fold hurts usability.
How: Move the analytics panels into a collapsible "Deal Analytics" accordion below the deal table. Default to collapsed. This puts the pipeline cards → deal table above the fold, matching the "see the situation → act on it" flow that makes the Overview tab effective.
5. Connect the deal tab to the coaching workflow (Medium Impact, Medium Effort)
Why: The Overview tab has checkboxes, action modals, and email generation. The deal tab is view-only. If Ramy spots a problem (AA has 5 deals stuck in Active Pipeline with zero offers), she has to navigate away to the Overview or Emails tab to do anything about it. Every tab-switch is friction.
How: Add a small "Coach" button on each AA's row in the deal table. Clicking it opens the email preview with the root cause pre-filled from the pipeline data: "Zero offers BECAUSE Active Pipeline deals have no Offer Terms or Submit Contract activity." This closes the loop between seeing the pipeline and driving action.
